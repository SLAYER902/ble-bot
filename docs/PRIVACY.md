# Privacy and retention

BLE Bot stores only data required for its enabled features: guild configuration, security evidence, cases, tickets, backup structure, reminders, and AI accounting. It does not back up deleted message history or webhook tokens.

Use retention controls before enabling high-volume modules. AI requests should contain only necessary content and must not include secrets or unrelated guild data. Guild deletion must be owner-authorized, confirmed, cancel scheduled work, remove storage objects, and retain only operationally necessary records.
