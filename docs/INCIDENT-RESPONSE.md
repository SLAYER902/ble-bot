# Incident response

1. Confirm the BLE Shield state and preserve the incident timeline.
2. Verify actor correlation and role hierarchy before any manual punishment.
3. Keep the guild in containment or lockdown while destructive activity continues.
4. Create a pre-restore snapshot and inspect a restore plan before applying recovery.
5. Record failed actions, evidence, and approvals.
6. Mark the incident resolved or false positive only after review.

Never punish an unresolved actor from a single gateway event. Never attempt to punish the guild owner. Do not use mass overwrite changes as an emergency response.
